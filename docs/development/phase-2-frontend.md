# Phase 2 - Frontend

## هدف فاز

ایجاد یک اسکلت اولیه‌ی فرانت‌اند مناسب برای نمایش وضعیت پروژه و آماده‌سازی مسیرهای بعدی برای ادغام با API.

## امکانات اجرا شده

- ایجاد صفحه‌ی اصلی با متن معرفی پروژه
- راه‌اندازی layout عمومی برای برنامه
- اضافه کردن سبک‌های پایه برای نمایش محتوای صفحه
- آماده‌سازی ساختار اولیه برای توسعه‌ی UI در فازهای بعدی

## فایل‌های ایجاد شده

- apps/web/src/app/page.tsx
- apps/web/src/app/layout.tsx
- apps/web/src/app/globals.css
- apps/web/next-env.d.ts
- apps/web/next.config.js

## فایل‌های تغییر داده شده

- apps/web/package.json
- apps/web/tsconfig.json

## تصمیم‌های معماری

- استفاده از App Router در Next.js و نگه‌داشتن کامپوننت‌ها به‌صورت Server Component در حد امکان
- محدود کردن UI اولیه به یک اسکلت ساده برای جلوگیری از پیچیده‌سازی زودهنگام
- حفظ تمرکز بر قابلیت توسعه در آینده و عدم ورود به مدیریت state پیچیده در این مرحله

## وابستگی‌های اضافه شده

- next
- react
- react-dom
- typescript
- eslint
- eslint-config-next

## دستورات اجرا شده

- pnpm --filter @castaminofen/web build

## نتایج اعتبارسنجی

- build فرانت‌اند با موفقیت انجام شد و برنامه به‌صورت موفق تولید شد.

## محدودیت‌های شناخته شده

- هیچ UI واقعی برای احراز هویت و پخش پادکست در این فاز وجود ندارد.
- سیستم ترجمه و design system هنوز پایه‌گذاری نشده‌اند.

## مرحله‌ی پیشنهادی بعدی

- ساخت shell کاربری و اتصال صفحه‌ی ورود/ثبت‌نام به API احراز هویت
