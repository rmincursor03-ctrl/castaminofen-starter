# گزارش فاز 2.4.1 — Frontend Integration & Auth Flow

## هدف

اتصال و تکمیل جریان احراز هویت، حفاظت مسیرها، و اتصال UI به عملیات CRUD پادکست و اپیزود در frontend فعلی.

## حوزه کاری

- بررسی و تثبیت providerهای React Query و Zustand
- تکمیل جریان auth و وضعیت session
- افزودن route protection برای مسیرهای مدیریت
- اتصال فرم‌های CRUD پادکست و اپیزود به API
- تکمیل UI آپلود فایل صوتی و نمایش وضعیت آن
- بررسی build و lint

## برنامه اجرایی

1. بررسی provider React Query و وضعیت storeهای موجود.
2. ایجاد یک store ساده برای auth state و استفاده از آن در صفحات مرتبط.
3. افزودن guard برای مسیرهای نیازمند auth.
4. تکمیل فرم‌های create/edit podcast و episode با هماهنگی با API.
5. بهبود UI upload audio و نمایش پیام‌های وضعیت.
6. اجرای build و lint و رفع خطاهای مرتبط.

## وضعیت پیشرفت

- زیرساخت React Query در حال استفاده است.
- Zustand store برای player موجود است و برای جریان auth تکمیل می‌شود.
- صفحات auth/podcast/episode در مرحله‌ی اولیه پیاده‌سازی شده‌اند و اکنون با flow واقعی‌تر هماهنگ می‌شوند.

## نکات مهم

- از افزودن معماری جدید خودداری شده و فقط با ساختار فعلی هماهنگ شده است.
- تلاش شده است تا از state‌های سرور در Zustand پرهیز شود و React Query برای داده‌های API استفاده شود.
- مجوز نهایی همچنان در backend بررسی می‌شود.
