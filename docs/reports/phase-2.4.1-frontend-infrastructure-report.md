# گزارش فاز 2.4.1 — Frontend Infrastructure

## هدف

تکمیل زیرساخت‌های اصلی frontend برای Castaminofen با تمرکز بر React Query provider، Zustand stores، لایه‌ی API client و تایپ‌های مشترک به‌گونه‌ای که صفحات فعلی و فازهای بعدی بتوانند بر پایه‌ی آن توسعه یابند.

## حوزه کاری

- راه‌اندازی و بهبود provider React Query
- تکمیل storeهای Zustand برای auth و player
- تقویت لایه‌ی API client با مدیریت خطا و هدرهای استاندارد
- انتقال تایپ‌های اصلی frontend به سطح مشترک و قابل استفاده در وب و بسته‌های مشترک
- بررسی وابستگی‌ها و ارزیابی build/lint

## برنامه اجرایی

1. بازبینی ساختار فعلی provider و stores در apps/web.
2. بهبود تنظیمات React Query با default options مناسب.
3. افزودن stateهای کاربردی به stores auth/player برای استفاده‌ی بهتر در صفحات.
4. تکمیل لایه‌ی API با ApiError و parsing امن‌تر برای پاسخ‌ها.
5. یکپارچه‌سازی تایپ‌های frontend با بسته‌ی shared-types.
6. اجرای build و lint و رفع خطاهای مرتبط.

## وضعیت پیشرفت

- provider React Query با تنظیمات پایه‌ی قابل استفاده تکمیل شد.
- store auth پشتیبانی از وضعیت authenticated و hydrated دارد.
- store player شامل کنترل‌های پایه برای volume، repeat و shuffle شده است.
- لایه‌ی API با مدیریت خطای استاندارد و parsing دقیق‌تر پیاده‌سازی شد.
- تایپ‌های اصلی به بسته‌ی shared-types منتقل و از آن در وب استفاده می‌شود.
- یک خطای تایپی در صفحه‌ی ایجاد اپیزود رفع شد و build/lint مجدداً بررسی شد.

## نتایج اعتبارسنجی

- نصب وابستگی‌ها با pnpm انجام شد.
- build وب با موفقیت اجرا شد بعد از رفع خطای تایپ.
- lint وب بدون خطا اجرا شد.

## نکات مهم

- این زیرفاز بر روی زیرساخت‌های MVP تمرکز دارد و بدون افزودن پیچیدگی غیرضروری انجام شده است.
- ساختار فعلی حفظ شده و برای توسعه‌ی فازهای بعدی آماده شده است.
